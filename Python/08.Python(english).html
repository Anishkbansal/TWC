This will be a short lecture so without any time waste lets get straight into the lecture.
Now, Python is an open source language which means that anyone can view the source code.now over the years people have wrote code in python and made that code public for other people to use
we call them libraries. a python library is some code written my some person which you can use for your own efficiency.

there are many types of libraries. some are inbuilt and some are not. the inbuilt libraries comes with the python package when you download python. others can be downloaded using pip

Peepo: 
pip is a software that downloads any library into your python. well how to use it? 
in the terminal, all you have to do is write        pip install <name of the library you want to install> <another library after the space if you want to install>   if multiple libraries at once, seperate the names with space     
and it will be installed.    you can also specify the version you want to download by typing pip install <name>==<version>

Peepo: 
now lets suppose you write a very good program which also uses a wide variety of libraries. now, you want to share your program to your friend.
there a strong chance that your friend might not have all the libraries you used to write program. he will get error and laugh at you.
and installing those libraries one by one is hectic. so what we do is we can create a text file. name it anything we want.

now in that text file we will put all the libraries we used. and the use    pip install -r <path to the file we created>.txt       all the libraries that arre written in txt file will be downloaded
# usually people name it requirements.txt

now suppose you wrote a code wayyy back in time and you are trying to run that code. but you forgot that the file uses older versions of libraries. which got recently updated. 
Now you would not really modify the code as per the newer version. so what you do is while writing some commercial code, you create a virtual enviroment which is like a seperate storage box which will store your libraries and is not affected by what is installed in your Global Python.
how do we create that enviroment?

Peepo: 
in the terminal type       python -m venv < path >     replace < path >  with what path of your file and name you want to give to your virtual env.

it is now created. remember virtual enviroment uses different library than global python which mean python installed in your computer that works without virtual enviroment.
so if you have any library installed in global python, you have to reinstall it in your virtual enviroment if you want it to work.

Peepo: 
so we have created our virtual enviroment now lets activate it to let python know that we are using it for this project
you just need to type.    <venv>/Scripts/anctivate.bat     or     <venv>/Scripts/anctivate.ps1         remember to replace <venv> with name of your virtual enviroment.

Peepo: 
now we are done with almost everything but how do we actually use these libraries. 
for that here comes the import statement. you type it like        import <name>        replace <name> with name of the library. 
you can also give the imported library a variable name. by using      import <name> as <variable_name>    replace <variable_name> with name of your variable

for example.. 
import pandas as pd         pnadas is name of library and pd is a variable

etcetera.
