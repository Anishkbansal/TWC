good. we use *args and **kwargs syntx. (with the stars).
*args (non-keyword argument)
The *args syntax allows you to pass an arbitrary(means random) number of non-keyword arguments to a function. The arguments are collected into a tuple within the function.



def func_args(*args):
    print(args , type(args))

func_args(1, 2, 3)  
func_args("hello", True, [1, 2, 3]) 





            In the above example, the print_args function can accept any number of arguments, and they are collected
            into the args tuple.
            You can also use *args in combination with other regular argument

            OUTPUT:

            
(1, 2, 3) <class 'tuple'>
('hello', True, [1, 2, 3]) <class 'tuple'>




            **Kwargs (keyword arguments)
            The **kwargs syntax allows you to pass an arbitrary number of keyword arguments to a function. The arguments
            are collected into a dictionary within the function, where the keys are the argument names, and the values
            are the argument values
            

def func_kwargs(**kwargs):
    print(kwargs , type(kwargs))

func_kwargs(name="Peepo", age=2)
func_kwargs(city="Love", country="Peepo_Heart", population="number of users of this app")






            OUTPUT:

            
{'name': 'Peepo', 'age': 2} <class 'dict'>
{'city': 'Love', 'country': 'Peepo_Heart', 'population': 'number of users of this app'} <class 'dict'>



            In this example, name is a regular argument, *args collects the non-keyword arguments (1, 2, 3) into a
            tuple, and **kwargs collects the keyword arguments (age=30, city="London") into a dictionary.
            Using *args and **kwargs allows you to write functions that can handle a flexible number of arguments, which
            can be useful in various situations, such as when dealing with user input or when working with functional
            programming techniques like higher-order functions.

lets see decorators. what are decorators in Python?  they are used to modify the function. for example
i have a function hello which prints hello. but when i run it i want good morning to be printing before the function starts. well i can do it manually in hello function but what if somewhere i want only hello to print? 
basically decorator function takes a function as argument and returns modified version of it

def goodmorning(fx): # giving function as argument
    def modify_fx(): # creating a function to modify the function
        print("goodmorning")   
        fx()     # running the function provided in argument
        print("function was successfuly modified")
    return modify_fx

@goodmorning      # syntax for decorator
def hello():
    print("hello")

hello()    # calling the hello function
